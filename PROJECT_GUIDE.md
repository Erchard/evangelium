# PROJECT GUIDE

## Швидка Навігація

| Завдання | Файл або папка |
| --- | --- |
| Редагувати український пролог | `book/front-matter/prologue-uk.md` |
| Редагувати англійський пролог | `book/front-matter/prologue-en.md` |
| Редагувати український основний текст | `book/text/logia-uk.md` |
| Редагувати англійський основний текст | `book/text/logia-en.md` |
| Редагувати український додаток | `book/appendix/commentary-uk.md` |
| Редагувати англійський додаток | `book/appendix/commentary-en.md` |
| Дивитися коптський шар | `book/translations/coptic.md` |
| Дивитися грецький шар | `book/translations/greek.md` |
| Дивитися картки логій | `research/logion-cards/` |
| Дивитися рішення включення | `research/decisions/` |
| Дивитися доказовий апарат | `research/evidence/` |
| Дивитися першоджерела | `sources/primary_texts/` |
| Запустити перевірку | `python build/scripts/qa_crosscheck.py` |
| Зібрати книжкові джерела | `python build/scripts/assemble_full_book_sources.py` |
| Дивитися готові файли | `dist/` |

## Що Є Чим

`book/` - це книга. Тут лежить те, що редагується як майбутній читацький текст.

`research/` - це дослідницька лабораторія. Тут картки, докази, рішення, аудити й методологія.

`sources/` - це збережені джерела, snapshots і матеріали, на які спирається дослідження.

`build/` - це механізм. Тут скрипти, які перевіряють і збирають проєкт.

`dist/` - це результат. Його не треба редагувати вручну.

`archive/` - це історія роботи. Вона збережена, але не має заважати щоденній навігації.

## Правило Для Майбутніх Змін

Новий текст книги йде в `book/`.

Нове джерело йде в `sources/`.

Нове дослідницьке рішення, картка або evidence note йде в `research/`.

Новий скрипт іде в `build/scripts/`.

Новий згенерований файл іде в `dist/`.

Старі промпти, проміжні звіти й одноразові аудити йдуть в `archive/project-history/`.
